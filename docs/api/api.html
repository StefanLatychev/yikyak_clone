API Documentation

YikYak Clone API uses the principle of RESTful API. HTTP status codes are returned on all API responses.

Usage

All api can be accessed via http requests to the appropriate api file. The type of http request made will depend of what type of request will be made. Http request types govern the following types of actions:

HTTP Request typeFunctionDescription
GETretrievesearch/fetch item(s)
PUTupdateupdate an item
POSTcreatecreate a new item
DELETEdeleteremove item

Each API request will have a parameter format indicated on in it's documentation for the its specific parameters. All formatted parameters are passed through an http request parameter.

Note that using the API requires authentication (in some cases), and a cookie must be sent every request to confirm authentication status.

Authentication

Server requires an https connection, and cookies enabled to authenticate and accept requests.

If a status of 401 unauthorized is received in response to a request, your API session key is invalid and you need to authenticate (potentially again).
